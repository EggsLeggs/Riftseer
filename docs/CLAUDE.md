## Overview

The docs site is a Bun workspace member. Run `bun install` from the repo root to install all dependencies, or from within `docs/` directly.

## Running

```bash
# From the repo root (preferred):
bun run --filter docs start   # dev server at http://localhost:3001 (avoids API on :3000)
bun run --filter docs build   # production build â†’ docs/build/

# Or from within docs/:
cd docs
bun install       # install dependencies (first time / after updates)
bun run start     # dev server at http://localhost:3001 (avoids API on :3000)
bun run build     # production build â†’ docs/build/
bun run serve     # serve the production build at http://localhost:3001
bun run clear     # clear Docusaurus cache â€” use if pages fail to update
```

## Where Doc Content Lives

Content is co-located with its package, **not** inside `docs/`. The Docusaurus config reads content from:

| Navbar section | Source path | URL prefix |
| --- | --- | --- |
| Getting Started | `docs/doc-pages/getting-started/` | `/getting-started/` |
| Ingest worker | `packages/ingest-worker/docs/` | `/` |
| Clients & Bots | `docs/doc-pages/clients-bots/`; Discord/Reddit pages are authored in `packages/*/docs/` and copied in by `sync-clients-bots-docs` (runs on `prestart` / `prebuild`) | `/bots/` |
| Core | `packages/core/docs/` | `/core/` |
| Types | `packages/types/docs/` | `/types/` |
| Supabase | `supabase/docs/` | `/supabase/` |

## Adding a Page

1. Create a `.md` file in the relevant source directory.
2. Add frontmatter:

   ```md
   ---
   title: My Page Title
   sidebar_label: Short Label
   ---
   ```

3. Sidebars are autogenerated â€” no extra config needed unless you want manual ordering (`sidebar_position`).

## Adding a New Section

When a new package needs documentation:

1. Create `packages/<name>/docs/*.md` with the content.
2. Add a plugin in `docs/docusaurus.config.ts`:

   ```typescript
   ['@docusaurus/plugin-content-docs', {
     id: 'section-id',
     path: '../packages/<name>/docs',
     routeBasePath: 'section-id',
     sidebarPath: './sidebarsSection.ts',
   }]
   ```

3. Create `docs/sidebarsSection.ts` (autogenerate pattern â€” copy `sidebarsSupabase.ts`).
4. Add a navbar item under `themeConfig.navbar.items` in `docusaurus.config.ts`.

## Mermaid Diagrams

## Deployment

Deployed to GitHub Pages via `.github/workflows/docs.yml`. The workflow runs `bun run build` inside `docs/` and publishes `docs/build/` to the `gh-pages` branch.

To deploy manually:

```bash
cd docs
bun run build
bun run deploy   # requires GH_TOKEN or SSH access to the repo
```

Update `url` and `organizationName` in `docusaurus.config.ts` to match your actual GitHub org/username before deploying.
